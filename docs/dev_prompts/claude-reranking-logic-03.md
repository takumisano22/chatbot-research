# 構造化チャンキング用再ランキングロジック作成プロンプト

## 役割
- あなたは、再ランキングのロジックに詳しく、作業者のコード実装をサポートするエージェントです。

## 目的
既存のreranking_logic_02では、親まで統合してしまうことで、長文により推論LLMの精度が下がることが分かりました。したがって、孫→子チャンクの統合は残して、親チャンクへの統合を削除した版を、reranking_logic_03.pyとして作成したいです。

## 要件
- 孫チャンクを子チャンクに統合するプロセスは残してください。
- 親チャンクは子チャンクが単一の場合は子チャンクを兼ねることもあります。その場合は、前項の孫チャンクの統合対象として既存の条件を満たしたら統合してください。その場合は、親へ子チャンクのようにスコアの引継ぎもしてよいです。
- 親チャンクへの統合プロセスは削除してください。親へのスコアの引継ぎも削除してください。
- その他のロジックは変えないでください。結果的にスコアの足きりを超えて親チャンクが残るのならそれは構いません。
- 実装した内容を、docsとしてstructure_aware_logic06_rerank02_guide.mdをstructure_aware_logic06_rerank03_guide.mdとして新規作成して更新する形で残してください。
