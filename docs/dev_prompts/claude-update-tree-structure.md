### チャンク分割：孫までの拡大方針
現在は、子チャンク分割までを行っていますが、孫を最小単位とするチャンク分割を実装してください。
    - 既存の親チャンク・子チャンクはそれぞれ残してください。
    - 親チャンクからマークダウンによる構造ツリーを使って子チャンクを抽出するのと同じやり方で、"####"を利用して孫チャンクを切り出してツリーに含んでください。
    - 孫チャンクは必ずしも存在しない（すべての子チャンクに####があるとは限らない）ので、なければ特に生成しなくていいです。
    - 既存の他にリスト記法がなければ"####"をリスト記法に置換することは、孫チャンクでも実装してください。
    - metadataについて、親は親チャンクid、子は親チャンクidと子チャンクid、孫は親チャンクidと子チャンクidと孫チャンクidを持てるようにしてください。

### 重複したチャンク分割切り出しの非実施
例えば、親チャンク内に子チャンクが一つしかない場合、親チャンクと子チャンクがほぼ等しくなってしまいます。このときチャンク保存を避けたいです。
    - ツリー構造には影響を与えず、下層チャンクが一つだとしてもツリーには含んで孫までたどってください。
    - チャンクの切り出しのタイミングで、一つ下層の子（もしくは孫）が一つの場合は、その切り出しは上層側一つですませて子では行わないで良いです。処理の流れ的に、親側は行う、子側は行わないとするのがスムーズかと思います。
    - ただし、子が一つしかなくて保存を行わなくても、孫側が複数ある可能性があるので、探索はして孫が複数なら孫チャンクは作成してください。
    - metadataの整備も行います。子チャンクまたは孫チャンクを省いた場合、上層側が下層チャンクのmetadataも兼ねるようにしてください。
