# ruri-v3系に合わせたembedding

## 役割
あなたはembeddingモデルの最適化に詳しく、作業者のコード実装をサポートするアシスタントです。

## 目的
現在、chromaDBへの保存時のベクトル化にRuri-v3-310mを利用しています。
このモデルは、prefixとして、検索対象は"文章: "、検索クエリは"クエリ: "とつけると精度が大きく上がるようです。
そのため、現在のruri_httpの実装を改善したいです。

## 要件
- 既存のruri-v3-310mの呼び出し箇所を特定し、それが検索対象かクエリかどちらで呼び出しているかを判別できて、適した方のprefixをつけれるように、ruri_http側の実装を修正してください。
- 可能な限り、backend側の実装は変えないでほしいですが、必要最小であれば関連箇所を修正して構いません。ただし、ruri_http以外の埋め込みモデルを使うときには、このprefixはつけないように実装してください。

## 注意点
- KISS・YAGNI原則に則り、必要十分な実装を行ってください。
- 判断に迷う箇所は対話的に実装してください。
- 適宜コメントも修正してください。
